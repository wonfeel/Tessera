FieldEngine
===========

Мой пет-проект на C++. Это маленький 2D-движок, который гоняет клеточные
автоматы (ну типа "Игра Жизнь") на поле, разбитом на куски-чанки, и рисует это
через OpenGL.

Делал в первую очередь чтобы научиться, а не чтобы это куда-то шло в продакшн.
Хотелось на практике разобраться с тем, что по книжкам плохо заходит:
- как раскидать симуляцию и рисование по разным потокам, чтобы не лагало
- как написать свой пул потоков и не словить гонки
- реально ли посчитать то же самое на видеокарте (CUDA) и насколько это быстрее

Это не первый мой заход в графику, до этого были:
- поле цветов прямо в консоли (с мышкой через WinAPI)
- первый рендер на OpenGL с нуля (шейдеры, буферы)
- первая кривая попытка "движка" без чанков и потоков
а потом уже вот это.


Что умеет
---------
- поле разбито на чанки, считаются только живые/видимые
- симуляция в одном потоке, рисование в другом - картинка не тормозит
  от тяжёлого расчёта (контекст OpenGL гоняю между потоками, это была самая
  нервная часть)
- свой пул потоков, чанки считаются параллельно, двойная буферизация чтобы
  рендер не поймал недосчитанный кадр
- правило автомата лежит таблицей, а не в коде - поэтому одно и то же правило
  работает и на CPU, и на GPU без переписывания
- бэкенд расчёта подменяемый: CPU или CUDA, выбирается на старте


Как собрать (Windows, Visual Studio + Ninja)
--------------------------------------------
  cmake --preset x64-release
  cmake --build out/build/x64-release

Дальше в out/build/x64-release/ появятся демки:
  Demo_minimallife_demo  - сама "Жизнь"
  Demo_fractal_demo      - фрактал Мандельброта (на том же движке, просто другой
                           источник данных - значит движок не прибит гвоздями
                           к автоматам)
  Demo_benchmark_demo    - меряет скорость

Библиотеки (GLFW, GLAD, GLM) уже лежат в libs/, ставить ничего не надо.


Про CUDA
--------
Если в системе есть CUDA Toolkit, cmake сам подхватит GPU-бэкенд:
  -- CUDA found - GPU simulation backend enabled
Если CUDA нет - спокойно собирается на CPU:
  -- CUDA not found - building CPU-only simulation backend
Выключить руками: -DFE_USE_CUDA=OFF


Скорость
--------
  Demo_benchmark_demo 1024 200

CPU на моей машине - примерно 215 Mcells/s (чанк 1024x1024, release, усреднил по
нескольким прогонам). Цифры по GPU пока нет, причина ниже.


Что ещё не доделано (честно)
----------------------------
- CUDA ещё не проверял на реальной видяхе - код есть и компилится, но toolkit у
  меня пока не установлен, так что сравнения CPU vs GPU нет. Это следующий шаг.
- когда GPU заработает, он будет гонять данные туда-сюда между CPU и видяхой
  каждый шаг, а это упирается в шину (PCIe). Хочу держать поле прямо в памяти
  GPU и рисовать оттуда (CUDA-OpenGL interop).
- в ChunkedTileMap интерфейс получился широковатый, надо разнести аккуратнее
- тестов почти нет
